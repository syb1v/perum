# PERUM — Multi-tenant School SaaS Platform

> Платформа Экономико-Аналитического Развития Учащейся Молодёжи. Геймифицированный электронный журнал с системой ливок, биржей, маркетом, квестами и аналитикой. Этот репозиторий — полный rewrite предыдущей версии (находящейся в `/Рабочий стол/PERUM`) с новой архитектурой: **silo-per-organization** + control plane.

## Что внутри

Монорепо из трёх независимо деплоящихся сервисов плюс инфраструктура:

| Каталог | Назначение |
|---|---|
| `perum-core/` | Control Plane. Один экземпляр на `admin.perum.ru`. Управляет организациями, провижинит docker-стеки, валидирует кастомные домены для Caddy, собирает метрики, ведёт биллинг. |
| `perum-tenant/` | Tenant App. Единый Docker-образ, запускается по одному инстансу на организацию (silo). Внутри инстанса — несколько школ организации, изолированных через `school_id`. |
| `perum-web/` | Next.js фронтенд. Один билд, multi-tenant: знает свою орг из subdomain или кастомного домена. Маршрутизирует UI: control plane (`admin.perum.ru`) vs tenant (`*.perum.ru` и кастомные домены). |
| `deploy/` | Caddy-шаблоны, docker-compose шаблоны per-org, Prometheus/Grafana конфиги, скрипты бэкапа. |
| `docs/` | Архитектура, изоляция тенантов, провижининг, RBAC, доменная политика. Стартовая точка для любого нового контрибьютора. |

## С чего начать

Прежде чем что-либо менять — прочитать в указанном порядке:

1. [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) — общая картина: что такое control plane, tenant app, как они общаются.
2. [docs/TENANT_ISOLATION.md](docs/TENANT_ISOLATION.md) — главный инвариант системы: одна орг = один процесс = одна БД. Понимать это до того, как писать новый модуль.
3. [docs/ROLES.md](docs/ROLES.md) — иерархия ролей (platform_admin → org_admin → school_admin → teacher/student/parent) и матрица прав.
4. [docs/PROVISIONING.md](docs/PROVISIONING.md) — как создаётся новая организация и её стек.
5. [docs/DOMAINS.md](docs/DOMAINS.md) — поток subdomain и кастомного домена через Caddy on-demand TLS.
6. [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md) — как раскатывать новую версию tenant-образа на все живые орг.
