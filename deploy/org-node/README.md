# Узел организации — развёртывание из коробки

Так поднимается **сервер организации** (узел орг) на чистой машине. Узел сам
провижинит школьные стеки (контейнер на школу) и обновляет их «по воздуху» по
кнопке — без доступа к серверу извне. См. [../../docs/ARCH_ORG_NODE.md](../../docs/ARCH_ORG_NODE.md).

## Шаги (минимум настройки)

1. **На сервере орг:** установить Docker + Docker Compose.
2. **В ядре (platform_admin):** создать организацию и выдать enrollment-токен:
   ```bash
   # организация (если ещё нет)
   curl -X POST https://<core>/api/organizations -H "Authorization: Bearer <PLATFORM_JWT>" \
     -H 'Content-Type: application/json' -d '{"slug":"acme","name":"Acme"}'
   # одноразовый токен подключения узла
   curl -X POST https://<core>/api/organizations/acme/enrollment-token \
     -H "Authorization: Bearer <PLATFORM_JWT>"
   #  → { "token": "...", "org_slug": "acme", "core_url": "...", "expires_at": "..." }
   ```
3. **На сервере орг:** запустить узел одной командой:
   ```bash
   CORE_URL=https://<core> ENROLLMENT_TOKEN=<token> \
     docker compose -f docker-compose.yml up -d
   ```
   Агент при старте предъявляет токен на `POST /api/enroll`, получает `org_slug` и
   текущий релиз, поднимает портал орг. Дальше `org_admin` входит в портал и
   создаёт школы — каждая в свой изолированный стек на этом же сервере.

## Обновления «по воздуху»

Платформа публикует релиз (тег образа + changelog). В портале орг у каждой школы
появляется кнопка «Обновить» → агент локально делает `docker pull` нового образа и
пересоздаёт **только** app-контейнер школы (БД и том целы) + миграции; при сбое —
автоматический откат. Никакого SSH и доступа к серверу не требуется.

## Статус реализации

- ✅ Выдача enrollment-токена (`POST /api/organizations/{slug}/enrollment-token`).
- ✅ Handshake `POST /api/enroll` (валидирует токен, отдаёт конфигурацию узла).
- ✅ Провижининг школьных стеков + OTA-обновления (в текущей сборке агент = ядро
  на одном хосте; `docker-compose.yml` здесь — reference-шаблон узла на отдельном
  сервере).
- ⬜ Вынесение агента в отдельный процесс с enroll-on-boot и удалённой регистрацией
  (структурный шаг для мульти-серверного деплоя; на одном хосте не требуется).
